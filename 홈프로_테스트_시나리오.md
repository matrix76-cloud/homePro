# 홈프로 테스트 시나리오

> 아래 순서대로 테스트해 주시면 됩니다.
> 테스트 시 **브라우저 2개**(또는 시크릿 창)를 열어 **접수자 / 홈프로 역할**을 나눠서 진행합니다.

---

## 0. 사전 준비

### 회원가입 (계정 2개 필요)
1. 홈프로 웹 접속
2. **계정 A** (접수자 역할) — 회원가입 → 로그인
   - 닉네임, 전화번호 등록
   - 지역 설정 (예: 충남 논산시)
3. **계정 B** (홈프로 역할) — 회원가입 → 로그인
   - 닉네임, 전화번호 등록
   - 지역 설정 (계정 A와 동일 지역)
   - **비즈프로필 → 전문분야 등록** (예: 이사.입주청소) → 관리자 승인 필요

### 관리자 승인
- 관리자 페이지에서 계정 B의 전문분야를 **승인** 처리
- 승인 후 계정 B가 요청목록에서 오더를 볼 수 있음

---

## 1. 오더 등록 (접수자 — 계정 A)

### 1-1. 우선배정호출 오더 등록
1. 계정 A 로그인
2. 메인 → **요청목록** 탭 → 하단 **"+ 예약접수"** 버튼 클릭
3. 카테고리 선택 (예: 이사.입주청소)
4. 폼 작성:
   - 작업날짜, 시간, 주소, 상세내용 입력
   - 단가유형: 확정가 또는 견적협의
   - **홈프로 선택: "우선배정호출"** 선택
5. 접수 완료

### 1-2. 다중비교호출 오더 등록
1. 동일하게 오더 생성
2. **홈프로 선택: "다중비교호출"** 선택
3. 접수 완료

### 1-3. 지정배정 오더 등록
1. 동일하게 오더 생성
2. **홈프로 선택: "지정배정"** 선택
3. **지정할 홈프로 전화번호** 입력 (계정 B의 전화번호)
4. 접수 완료

---

## 2. 요청목록 확인 (홈프로 — 계정 B)

### 2-1. 메인 요청목록
1. 계정 B 로그인
2. 메인 → **요청목록** 탭
3. 방금 등록된 오더가 **"접수"** 상태로 보이는지 확인
4. 필터 버튼 (거리/상태/기간/카테고리) 동작 확인
5. **마감된 오더** 클릭 시 → 토스트 메시지 "이미 마감된 항목은 확인할 수 없습니다" 확인

### 2-2. 오더 상세 진입
1. 접수 상태 오더 클릭 → 상세 페이지 진입
2. 오더 정보 (카테고리, 지역, 날짜, 상세내용) 확인
3. **배정 전**: 전화번호, 상세주소는 보이지 않아야 함

---

## 3. 우선배정호출 테스트

### 3-1. 홈프로가 수락
1. 계정 B → 우선배정호출 오더 상세 진입
2. **"수락하기"** 버튼 클릭
3. 확인 → 바로 **배정** 상태로 변경됨
4. 메인 요청목록에서 해당 오더가 **"마감"**으로 표시되는지 확인

### 3-2. 접수자 확인
1. 계정 A → **나의오더현황** 탭
2. 해당 오더가 **"배정"** 탭에 있는지 확인
3. 배정된 홈프로 정보 (이름, 연락처 등) 확인 가능

### 3-3. 이미 배정된 오더
- 다른 홈프로가 같은 오더에 수락 시도 → **"배정된 오더입니다"** 안내 메시지

---

## 4. 다중비교호출 테스트

### 4-1. 홈프로가 지원
1. 계정 B → 다중비교호출 오더 상세 진입
2. **"지원하기"** 버튼 클릭
3. "지원이 완료되었습니다" 토스트 확인
4. (테스트 시 3명 채우기 어려우면, 1명 지원 상태에서 접수자가 수동 선정 가능)

### 4-2. 접수자가 업체 선정
1. 계정 A → **나의오더현황** → **"업체선택"** 탭
2. 해당 오더 클릭 → 지원자 목록 확인
3. 지원자 정보 확인: 상호, 이름, 프로필, 소개글, 리뷰
4. **"선정"** 버튼 클릭 → 해당 홈프로가 **배정**됨
5. 나머지 지원자에게 "다른 홈프로가 선정되었습니다" 알림

### 4-3. 3명 지원 시 자동 마감
- 3명이 지원 완료되면 메인 요청목록에서 **자동 마감** 처리
- 접수자는 업체선택 탭에서 1명 선정

---

## 5. 지정배정 테스트

### 5-1. 지정된 홈프로 확인
1. 계정 B → 지정배정 오더가 보이는지 확인
2. **"수락하기"** 또는 **"거절하기"** 버튼 확인
3. 수락 → **배정** 상태로 변경
4. 거절 → **대기** 상태로 복귀 (접수자가 재접수 가능)

---

## 6. 나의오더현황 테스트 (접수자 — 계정 A)

### 6-1. 상태 탭 확인
1. 계정 A → **나의오더현황** 탭
2. 상단 탭 7개 확인: 전체 / 등록 / 배정 / 업체선택 / 완료 / 취소 / 거부
3. 각 탭 클릭 시 해당 상태 오더만 필터링되는지 확인
4. 탭 아래 설명 텍스트 확인

### 6-2. 상태 변경 버튼 테스트
| 현재 상태 | 접수자 버튼 | 홈프로 버튼 |
|-----------|-------------|-------------|
| 등록 | 대기 / 취소 | - |
| 배정 | 취소 | 작업완료 |
| 업체선택 | 지원자 보기 | - |
| 대기 | 재접수 / 수정 | - |
| 완료 | (없음) | (없음) |

### 6-3. 취소 테스트
1. 등록 또는 배정 상태 오더에서 **"취소"** 클릭
2. 취소 사유 선택: 잘못접수 / 일정변경 / 오더수정 / 기타
3. 선택 후 취소 처리 확인

### 6-4. 대기 → 재접수 테스트
1. 등록 상태 오더 → **"대기"** 클릭 → 대기 상태로 전환
2. 대기 상태 오더 → **"재접수"** 클릭 → 다시 접수 상태로 (지원자 초기화)

---

## 7. 채팅 테스트

### 7-1. 채팅 진입
1. 지원/배정 단계에서 채팅 가능
2. 채팅방 상단 상태 버튼 5개 확인: 등록 / 배정하기 / 작업완료 / 대기전환 / 취소하기
3. 현재 상태에 보라색 표시, 변경 가능한 버튼만 활성화

### 7-2. 채팅에서 상태 변경
1. 접수자: **배정하기** 클릭 → 배정 확인
2. 홈프로: **작업완료** 클릭 → 완료 확인
3. 접수자: **취소하기** 클릭 → 사유 선택 바텀시트 → 취소 확인

### 7-3. 거부 등록
1. 채팅 상단 **더보기(...)** 메뉴 → **"거부 등록"**
2. 사유 입력 (자유 텍스트) → 등록 완료

---

## 8. 작업완료 → 리뷰 테스트

### 8-1. 작업완료 처리
1. 배정 상태 오더 → 홈프로(계정 B)가 **"작업완료"** 클릭
2. 바로 완료 처리 (접수자 승인 없음)

### 8-2. 리뷰 작성
1. 완료된 오더 → 접수자(계정 A)가 리뷰 작성
2. 별점 (5점) + 텍스트 후기
3. 리뷰는 **접수자 → 홈프로** 단방향만

---

## 9. 거부 / 블랙리스트 테스트

### 9-1. 거부 목록 관리
1. 마이페이지 → **차단 관리** → **거부 목록**
2. 거부 등록된 사용자 목록 확인
3. **"해제"** 버튼으로 거부 해제 가능
4. 거부된 상대의 오더: 보이되, 수락/지원 클릭 시 **"거부된 오더입니다"**

### 9-2. 블랙리스트 테스트
1. 계정 A → 계정 B의 **비즈프로필** 진입
2. 하단 **"블랙리스트 신고"** 버튼 클릭
3. 사유 입력 → 등록
4. 마이페이지 → **차단 관리** → **블랙리스트** 에서 확인
5. 블랙리스트 등록된 사람은 **나한테 오더 요청 불가**
6. **"해제"** 버튼으로 블랙리스트 해제 가능

---

## 10. 기타 확인 사항

### 필터 기능 (요청목록)
- 새로고침 / 거리 / 상태 / 기간 / 카테고리 필터 동작 확인

### 정보 공개 규칙
- **배정 전**: 카테고리, 지역(시/구까지), 날짜, 상세내용만 표시
- **배정 후**: 전화번호, 상세주소 추가 공개

### 자동 취소 (추후 테스트)
- 예약시간 기준 자동취소 → 접수자에게만 알림
