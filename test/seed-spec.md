## 10. 정리 (Cleanup) 정책

> ⚠️ **경고**: `cleanAllTestData`는 **운영/시드 구분 없이 화이트리스트 컬렉션 전체와 Auth 사용자 전체를 삭제**한다.
> 운영 데이터가 들어있는 Firebase 프로젝트에서는 절대 호출하지 말 것. dev 전용.

`cleanAllTestData` 호출 시:
- 화이트리스트 컬렉션 전체 wipe: `users`, `phones`, `homepro_orders`(+서브컬렉션 applicants/quotes/reviews/messages), `homepro_worker_requests`, `homepro_marketplace`, `homepro_trainings`, `homepro_cash`, `homepro_subscriptions`, `chatRooms`(+messages 서브컬렉션)
- Firebase Auth 사용자 전체 삭제 (`auth.listUsers` → `deleteUsers`)

> 모든 시드 문서에는 `_isSeed: true` 마커가 추가되지만, 현재 cleanup은 마커를 무시하고 전체 삭제한다.
> 향후 운영/시드 분리가 필요해지면 cleanup을 `_isSeed:true` 한정 또는 `seed_*` uid 한정으로 좁힐 것.
